**VocabProfiler-FR** 是一个**自用**的语言学习者词汇量化管理系统。不同于传统的单词记忆软件，本项目将词汇分为“认知词（Passive）”与“掌握词（Active）”，并引入双向记忆与复习队列控制，提供更符合记忆规律的学习流程。

### 1. 项目愿景

在语言学习中，学习者往往面临词汇增长缓慢、逻辑连接词调用延迟等“系统瓶颈”。本项目旨在通过信息化手段重构词汇学习流程：

- **随时记录**：随时记录和管理词汇。
- **科学记忆**：自动管理词汇生命周期与复习节奏。
- **量化反馈**：通过等级系统（Level 0-5）可视化记忆状态，让用户看到自己距离目标还有多远。

### 2. 技术栈 

项目采用前后端分离架构，确保系统的可扩展性与稳定性：

- **前端**: Vue.js (基于组件化开发，提供响应式移动端体验)
- **后端**: Node.js + Express.js (处理词汇生命周期逻辑与 API 路由)
- **数据库**: MySQL (持久化存储词汇数据、等级状态及复习周期)
- **工程规范**: 采用 Conventional Commits 提交标准，保持热力图的高质量贡献。

### 3. 主要功能

- **录入模块**

  快速录入新词，随时录入，不提供更多功能，只要

  - 卡片正面写上词汇
  - 选择属于看到认识的词汇还是双向都要会的词，只做这一个分类
  - 要记忆的解释，即卡片背面
  - 如果录入的词汇已经存在，那么直接修改已有的卡片背面，而不是录入，该词汇的level和复习时间将会被重置

- **复习与学习机制**

  词汇按类型区分为 Active 与 Passive：
  - **Passive**：只要求正向掌握。
  - **Active**：必须正向与反向都掌握，反向词的作用是辅助正向词升级，只有反向掌握后，正向掌握才会触发等级更新。

  不管 Active 还是 Passive，都分为两种模式：
  - **学习新词**（New）
  - **复习旧词**（Review）

  **复习模块（Review）**
  - 维护一个“今日复习队列”，根据 `NextReviewDate <= Today` 筛选待复习词库。
  - 复习阶段提供三种选择：**掌握 / 模糊 / 忘记**。
    1. **掌握**：
       - Passive 直接更新等级并移出复习队列。
       - Active 需要反向词也掌握，正向先暂存，待反向掌握时再统一更新。
    2. **模糊 / 忘记**：
       - 立即更新等级。
       - 复习队列中把该词移到队尾，进入“当日重复复习”。
  - 当“今日复习队列”走完后，会进入**重复复习界面**（界面与学习新词一致，但不再更新等级，只是巩固）。

  **学习新词（New）与记忆窗口**
  - 新词只提供两个按钮：**记住了 / 没记住**。
  - 引入“记忆窗口”，默认最多 5 个词：
    - 当窗口满时，新加入的词会顶出队头。
    - 被顶出的词视为完成记忆，触发后端等级更新。
  - 多义词处理：
    - 同一单词的多个解释会合并展示，作为一组进入队列与窗口。
    - 当一个多义词进入窗口时，会动态扩展窗口长度，确保同组词一起学习。
    - 被顶出时仍按“逐个解释”的节奏更新，允许额外重复以提高稳固性。

  **词汇生命周期：**
  - 新词被加入，会进入 level 0，意思是待学习，用重复机制，其他`level > 0`的词汇才是复习机制
  - 新词level 0学完后变到level 1，1天后复习
  - level 1复习完点击“掌握”后变成level 2，3天后复习
  - level 2复习完点击“掌握”后变成level 3，7天后复习
  - level 3复习完点击“掌握”后变成level 4，14天后复习
  - level 4复习完点击“掌握”后变成level 5，一个月后复习
  - 一个月后level 5复习也通过了，就认为已经掌握了这个词汇
  - 当中但凡有一个词汇点击了“忘记”，就重置这个词汇成为“level 0，1天后复习”的状态。“忘记”后当天这个待复习卡片会进入重复，直到你点击了“记住了”。

- **词库管理**

  - 可视化展示全部已存词汇，仅此而已，没有其他功能。第一个版本不支持修改、排序、筛选等。默认展示顺序按照添加日期排序

- **统计看板/设置**

  - 第一版不做看板，先保证可用
