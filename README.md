**VocabProfiler-FR** 是一个**自用**的语言学习者词汇量化管理系统。不同于传统的单词记忆软件，本项目将词汇分为“认知词（Passive）”与“掌握词（Active）”，利用科学的间隔复习算法（SRS），提供词汇记忆解决办法。

### 1. 项目愿景

在语言学习中，学习者往往面临词汇增长缓慢、逻辑连接词调用延迟等“系统瓶颈”。本项目旨在通过信息化手段重构词汇学习流程：

- **随时记录**：随时记录和管理词汇。
- **科学记忆**：自动管理词汇生命周期。
- **量化反馈**：通过等级系统（Level 0-5）可视化记忆状态，让用户看到自己距离目标还有多远。

### 2. 技术栈 

项目采用前后端分离架构，确保系统的可扩展性与稳定性：

- **前端**: Vue.js (基于组件化开发，提供响应式移动端体验)
- **后端**: Node.js + Express.js (处理词汇生命周期逻辑与 API 路由)
- **数据库**: MySQL (持久化存储词汇数据、等级状态及复习周期)
- **工程规范**: 采用 Conventional Commits 提交标准，保持热力图的高质量贡献。

### 3. 主要功能

- **录入模块**

  快速录入新词，随时录入，不提供更多功能，只要

  - 卡片正面写上词汇
  - 选择属于看到认识的词汇还是双向都要会的词，只做这一个分类
  - 要记忆的解释，即卡片背面
  - 如果录入的词汇已经存在，那么直接修改已有的卡片背面，而不是录入，该词汇的level和复习时间将会被重置

- **复习模块**

  - 分类2种复习，按照 ”认知词“ 和 ”双向词“ 区分成两块
  - 维护一个”今日复习队列“
  - 根据 `NextReviewDate <= Today` 筛选待复习词库。
  - 提供“掌握/模糊/忘记” ou “记住了/没记住”的交互逻辑，驱动后端状态机更新单词等级。
    1. ”重复“机制：如果是模糊/忘记，那么这一轮复习完后会重复让你去复习直到记住他，当天的重复复习“记住了”/“没记住”不会影响单词的等级和下次复习时间的确定，即把需要重复的词汇重新加入”今日复习队列“（回到队尾）
    2. 新词的学习：用重复机制，而不是 ”掌握/模糊/忘记“ 会重复让你去背诵直到记住他，这是第一遍记忆
    3. 第一次复习的选择：掌握/模糊/忘记 直接决定了单词的等级和下一次复习时间的确定。
       - 选择了掌握，直接过，增加时间，移出”今日复习队列“
       - 选择忘记，会在第一轮的”掌握/模糊/忘记“检测完后进入重复，并且在重复直到“记住了”之后才会结束。遗忘的词汇会回到level 0，下次复习时间变成一天后，重新加入”今日复习队列“
       - 选择模糊，会减少1级时间等级（最低是level 1不能在小了，level 0是新词专属等级），同样会触发重复。
  - 词汇生命周期：
    - 新词被加入，会进入 level 0，意思是待学习，用重复机制，其他`level > 0`的词汇才是复习机制
    - 新词level 0学完后变到level 1，1天后复习
    - level 1复习完点击“掌握”后变成level 2，3天后复习
    - level 2复习完点击“掌握”后变成level 3，7天后复习
    - level 3复习完点击“掌握”后变成level 4，14天后复习
    - level 4复习完点击“掌握”后变成level 5，一个月后复习
    - 一个月后level 5复习也通过了，就认为已经掌握了这个词汇
    - 当中但凡有一个词汇点击了“忘记”，就重置这个词汇成为“level 0，1天后复习”的状态。“忘记”后当天这个待复习卡片会进入重复，直到你点击了“掌握”，这一点和新词是一样的，所以变成 level 0，1天后复习 合理。

- **词库管理**

  - 可视化展示全部已存词汇，仅此而已，没有其他功能。第一个版本不支持修改、排序、筛选等。默认展示顺序按照添加日期排序

- **统计看板/设置**

  - 第一版不做看板，先保证可用
